# dotlink Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `bin/dotlink`, a bash CLI that replaces `install/link.sh` — providing bulk linking (`sync`) plus add/unlink of `~/.config/<tool>/` directories into dotfiles management.

**Architecture:** Single bash script `bin/dotlink` with a subcommand dispatcher (`sync`, `add`, `unlink`). The core linking primitive (`link_config`) is ported verbatim from `link.sh` and shared by `sync` and `add`. `sync` reproduces `link.sh`'s three-root scan; `add`/`unlink` operate only on `~/.config/<tool>/` directories.

**Tech Stack:** bash (macOS system bash 3.2 compatible — no associative arrays), git, standard coreutils (`ln`, `mv`, `mkdir`, `find`, `basename`).

## Global Constraints

- Language: bash, macOS system bash 3.2-safe (no `declare -A`, no `mapfile`).
- `DOTFILES=$HOME/.dotfiles`, `BACKUP_DIR=$HOME/.backup` (create with `mkdir -p` when needed).
- Target of `add`/`unlink` is `~/.config/<tool>/` **directories only** — single-file `link/*.symlink` entries are handled by `sync` alone.
- Emoji/message style follows existing `link.sh` (`✅`, `⚠️`, `🔗`, `❗️`, `👉`).
- Symlinks use `ln -sf`; "already managed" is detected with `-ef` (same inode) as in `link.sh`.
- `unlink` performs working-tree moves only — no git operations.
- Unknown subcommand / missing args → print usage, exit non-zero.

- [ ] **Step 1: Write the script skeleton**

Create `bin/dotlink`:

```bash
#!/usr/bin/env bash

DOTFILES="$HOME/.dotfiles"
BACKUP_DIR="$HOME/.backup"

usage() {
  cat >&2 <<'EOF'
dotlink — dotfiles link manager

Usage:
  dotlink sync [--force]      Link all dotfiles into $HOME and ~/.config
  dotlink add <tool> [--mac]  Adopt ~/.config/<tool>/ into dotfiles (default: dotconfig/, --mac: config.mac/)
  dotlink unlink <tool>       Remove <tool> from dotfiles, moving it back to ~/.config/<tool>/
EOF
}

# Ported verbatim from install/link.sh — uses global $force.
link_config() {
  config="$1"
  link_name="$2"
  echo "-> config: $config link_name: $link_name"
  if [ -e "$link_name" ]; then
    if ! [ "$link_name" -ef "$config" ]; then
      if [ "$force" = "force" ]; then
        echo "❗️${link_name} already exists, backup it and do link staff."
        mv "$link_name" "$BACKUP_DIR/"
        ln -sf "$config" "$link_name"
        echo "✅ Linked $config to $link_name"
      else
        echo "⚠️  ${link_name} already exists, skip."
      fi
    else
      echo "🔗 ${link_name} already linked."
    fi
  else
    ln -sf "$config" "$link_name"
    echo "✅ Linked $config to $link_name"
  fi
}

cmd_sync() { :; }    # implemented in Task 2
cmd_add() { :; }     # implemented in Task 3
cmd_unlink() { :; }  # implemented in Task 4

main() {
  subcommand="$1"
  shift 2>/dev/null
  case "$subcommand" in
    sync)   cmd_sync "$@" ;;
    add)    cmd_add "$@" ;;
    unlink) cmd_unlink "$@" ;;
    *)      usage; exit 1 ;;
  esac
}

main "$@"
```

Replace the `cmd_sync() { :; }` stub with:

```bash
cmd_sync() {
  force=""
  if [ "$1" = "--force" ]; then
    force="force"
    echo "!!!Run in force mode, be careful."
  fi

  [ ! -d "$BACKUP_DIR" ] && mkdir -p "$BACKUP_DIR"

  printf "\n👉 Linking files into home directory.\n"
  find -H "$DOTFILES/link" -maxdepth 3 -name '*.symlink' -print0 | while read -r -d $'\0' config; do
    link_name="$HOME/.$(basename "$config" '.symlink')"
    link_config "$config" "$link_name"
  done

  printf "\n👉 Linking files into ~/.config directory.\n"
  find -H "$DOTFILES/dotconfig" -maxdepth 1 -mindepth 1 -type d -print0 | while read -r -d $'\0' config; do
    link_name="$HOME/.config/$(basename "$config")"
    link_config "$config" "$link_name"
  done

  if [ "$(uname -s)" = "Darwin" ]; then
    printf "\n👉 Linking mac specific config into ~/.config directory.\n"
    find -H "$DOTFILES/config.mac" -maxdepth 1 -mindepth 1 -type d -print0 | while read -r -d $'\0' config; do
      link_name="$HOME/.config/$(basename "$config")"
      link_config "$config" "$link_name"
    done
  fi

  printf "\n✨ done.\n"
}
```

- [ ] **Step 1: Implement `cmd_add`**

Replace the `cmd_add() { :; }` stub with:

```bash
cmd_add() {
  tool=""
  subdir="dotconfig"
  for arg in "$@"; do
    case "$arg" in
      --mac) subdir="config.mac" ;;
      -*)    echo "⚠️  unknown flag: $arg"; usage; exit 1 ;;
      *)     tool="$arg" ;;
    esac
  done

  if [ -z "$tool" ]; then
    echo "⚠️  add: missing <tool>."; usage; exit 1
  fi

  src="$HOME/.config/$tool"
  dest="$DOTFILES/$subdir/$tool"

  # Already managed by dotfiles → idempotent skip.
  # -ef is true only when both paths exist and share an inode, so a
  # symlink src pointing into either root matches without extra guards.
  if [ "$src" -ef "$DOTFILES/dotconfig/$tool" ] || [ "$src" -ef "$DOTFILES/config.mac/$tool" ]; then
    echo "🔗 ~/.config/$tool is already managed by dotfiles, skip."
    return 0
  fi

  # Nothing to adopt.
  if [ ! -e "$src" ]; then
    echo "❌ ~/.config/$tool does not exist, nothing to adopt."; exit 1
  fi

  [ ! -d "$BACKUP_DIR" ] && mkdir -p "$BACKUP_DIR"

  if [ -e "$dest" ]; then
    # dotfiles already has this tool (adopted on another machine).
    echo "❗️$dest already exists in dotfiles, backing up local ~/.config/$tool."
    mv "$src" "$BACKUP_DIR/"
  else
    echo "📦 Moving ~/.config/$tool into $subdir/$tool."
    mv "$src" "$dest"
  fi

  ln -sf "$dest" "$src"
  echo "✅ Adopted ~/.config/$tool → $subdir/$tool."
}
```

Replace the `cmd_unlink() { :; }` stub with:

```bash
cmd_unlink() {
  tool="$1"
  if [ -z "$tool" ]; then
    echo "⚠️  unlink: missing <tool>."; usage; exit 1
  fi

  src="$HOME/.config/$tool"

  # Find the real directory inside dotfiles.
  if [ -d "$DOTFILES/dotconfig/$tool" ]; then
    real="$DOTFILES/dotconfig/$tool"
  elif [ -d "$DOTFILES/config.mac/$tool" ]; then
    real="$DOTFILES/config.mac/$tool"
  else
    echo "❌ $tool is not managed by dotfiles (no dotconfig/$tool or config.mac/$tool)."; exit 1
  fi

  # Remove the symlink if present.
  if [ -L "$src" ]; then
    rm "$src"
  elif [ -e "$src" ]; then
    echo "⚠️  ~/.config/$tool exists but is not a symlink; leaving it in place."
    echo "    Resolve manually, then rerun unlink."
    exit 1
  fi

  mv "$real" "$src"
  echo "✅ Unlinked $tool → moved back to ~/.config/$tool."
  echo "   Note: run 'git rm -r --cached' / commit in $DOTFILES to drop it from git."
}
```

## Notes for the implementer

- **bash 3.2:** macOS system bash is 3.2 — no associative arrays. This plan uses none; keep it that way.
- **`-ef` semantics:** true when two paths resolve to the same inode. `link.sh` relies on this to detect "already linked"; `dotlink` keeps the same idiom.
- **Why `sync` re-runs are safe:** every existing correct link hits the `-ef` branch and prints "already linked" — no move, no backup, no clobber. Run it freely during verification.
- **git hygiene in verification:** the `dotlink-test` fixtures live in `~/.config` and `dotconfig/`; the cleanup steps remove both so no test artifact is ever committed. Always confirm `git status` is clean of fixtures before the commit step.
